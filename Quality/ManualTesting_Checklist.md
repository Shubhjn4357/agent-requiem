# Manual Testing Checklist for Release Quality

## 1. Sanity Check

- Verify core application functionality (login, dashboard, key actions).

## 2. Regression Testing

- Ensure that new changes haven't broken existing, unrelated features.

## 3. Visual QA

- Check for UI inconsistencies, broken layouts, and visual regressions across different browsers and devices.

## 4. Edge Case Validation

- Test with empty states, long text inputs, and unusual user permissions.

## 5. Deployment Verification (Post-Release)

- Perform a quick manual smoke test in the production environment after every deployment.
