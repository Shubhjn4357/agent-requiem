# Internationalization (i18n) Standards

## Translation Strategy

1. **Centralized Keys**: Store all strings in `locales/en.json`, `locales/hi.json`, etc.
2. **Namespace Support**: Organize keys by feature (e.g., `billing.invoice.title`).
3. **Pluralization**: Use standard i18next plural rules.

## Implementation Guide

- **Hook-Based**: Use `useTranslation()` hook.
- **Dynamic Content**: Inject variables using interpolation (`{{name}}`).
- **Date/Currency**: Use `Intl` API or `date-fns` for locale-aware formatting.
- **RTL Support**: Ensure layout mirrors correctly for RTL languages (e.g., Arabic).
