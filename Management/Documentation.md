# Documentation Standards

## In-Code Documentation

1. **JSDoc**: Use for all complex functions, hooks, and types. Include `@param`, `@returns`, and `@example`.
2. **Self-Documenting Code**: Priority 1. Avoid comments that explain _what_ the code is doing—use them for _why_.
3. **READMEs**: Every major directory (`lib`, `hooks`, `components`) should have a local `README.md` explaining its purpose and contents.

## Markdown Guidelines

- Use headers (`#`, `##`) for structure.
- Fence all code blocks with proper language syntax (e.g., ` ```typescript `).
- Use checklists for progress and bold text for emphasis.
- Always provide relative or absolute file paths when referencing code.
