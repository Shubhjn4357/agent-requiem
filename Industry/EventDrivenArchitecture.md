# Event-Driven Architecture (EDA)

## 1. Description

Event-Driven Architecture is an architectural pattern in which events are used to communicate between different parts of the system.

## 2. Decoupling

Events allow for loose coupling between components, making the system more flexible and scalable.

## 3. Event Sourcing

Event sourcing is a common pattern used with EDA, where the current state of a system is derived from a sequence of events.

## 4. Message Brokers

Use message brokers like Kafka, RabbitMQ, or NATS to handle event distribution.

## 5. Benefits

- Improved scalability and availability.
- Increased flexibility and extensibility.
- Enhanced reliability and resilience.
