# Serverless Patterns

## 1. Description

Serverless patterns are architectural patterns used with serverless technologies like AWS Lambda or Google Cloud Functions.

## 2. Event-Driven Execution

Serverless functions are triggered by events, such as a file upload or a database change.

## 3. Scalability and Availability

Serverless functions are automatically scaled and highly available.

## 4. Short-Lived Functions

Serverless functions should be short-lived and perform a single task.

## 5. Security and Billing

Serverless functions should be designed to be secure and only run when needed to minimize costs.
