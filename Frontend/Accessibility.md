# Accessibility (a11y) in Web Development

## 1. Semantic HTML

Use the correct HTML elements for their intended purposes (e.g., `<button>` for buttons, `<nav>` for navigation).

## 2. ARIA Attributes

Use ARIA (Accessible Rich Internet Applications) attributes to provide additional context for screen readers when semantic HTML is not enough.

## 3. Keyboard Navigation

Ensure all interactive elements are accessible and usable via the keyboard alone.

## 4. Color Contrast

Provide sufficient color contrast between text and background to ensure readability for users with visual impairments.

## 5. Alternative Text

Provide descriptive alternative text for images and other non-text content.
